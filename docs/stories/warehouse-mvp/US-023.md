# US-023: Insufficient stock alert

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-020

---

## Goal

Avoid preparing unfeasible orders by informing the operator before reserving stock when there is not enough availability in the warehouse.

## User

**Operator** (`operario`), **Supervisor** (`supervisor`)

## Description

As an operator, I want to see a clear alert when the outbound order requests more quantity than is available in the warehouse, so I can decide whether to adjust quantities or postpone the order.

## Acceptance criteria

- [ ] When trying to move an order from `draft` to `picking`, the system validates available stock per line
- [ ] Available = sum(`quantity_on_hand - quantity_reserved`) per product in the warehouse
- [ ] If any line has a shortfall, show an alert with: product, requested, available, shortage
- [ ] Do not start picking automatically if there is a shortfall (blocked in the MVP)
- [ ] Supervisor can see the same shortfall detail in the order view
- [ ] Actionable message: edit quantities or cancel

## Business rules

1. Validation against the order's warehouse only (no cross-warehouse on outbound).
2. In-transit stock (transfers) does not count as available at the destination until receipt.
3. Inactive products do not add to availability.
4. No automatic backorder in the MVP.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Exact availability | Allow picking |
| Availability with decimals (kg) | Comparison with a tolerance of 0.001 |
| Stock freed by cancelling another order | Re-validate on retry |
| Multiple lines with the same product | Validate the combined sum vs availability |

## Preliminary technical notes

- Aggregate query: `StockLevel.where(warehouse_id:, product_id:).sum(:on_hand) - sum(:reserved)`
- `Outbound::AvailabilityChecker.call(order)` service → `Result` with shortfalls
- Show in a UI modal before `POST .../start_picking`
- Branch: `feature/warehouse-outbound`

## Versión en español

# US-023: Alerta de stock insuficiente

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-020

---

## Objetivo

Evitar preparar pedidos inviables informando al operario antes de reservar stock cuando no hay disponibilidad suficiente en el almacén.

## Usuario

**Operario** (`operario`), **Supervisor** (`supervisor`)

## Descripción

Como operario, quiero ver una alerta clara cuando la orden de salida solicita más cantidad de la disponible en el almacén, para decidir si ajustar cantidades o posponer el pedido.

## Criterios de aceptación

- [ ] Al intentar pasar orden de `draft` a `picking`, el sistema valida stock disponible por línea
- [ ] Disponible = suma(`quantity_on_hand - quantity_reserved`) por producto en el almacén
- [ ] Si alguna línea tiene déficit, mostrar alerta con: producto, solicitado, disponible, faltante
- [ ] No iniciar picking automáticamente si hay déficit (bloqueo en MVP)
- [ ] Supervisor puede ver mismo detalle de déficit en vista de orden
- [ ] Mensaje accionable: editar cantidades o cancelar

## Reglas de negocio

1. Validación contra almacén de la orden únicamente (no cross-almacén en salida).
2. Stock en tránsito (transferencias) no cuenta como disponible en destino hasta recepción.
3. Productos inactivos no suman disponibilidad.
4. Sin backorder automático en MVP.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Disponible exacto | Permitir picking |
| Disponible por decimales (kg) | Comparación con tolerancia 0.001 |
| Stock liberado por cancelación otra orden | Re-validar al reintentar |
| Múltiples líneas mismo producto | Validar suma conjunta vs disponible |

## Notas técnicas preliminares

- Query agregada: `StockLevel.where(warehouse_id:, product_id:).sum(:on_hand) - sum(:reserved)`
- Servicio `Outbound::AvailabilityChecker.call(order)` → `Result` con déficits
- Mostrar en UI modal antes de `POST .../start_picking`
- Rama: `feature/warehouse-outbound`
