# US-022: Confirm picking

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-021

---

## Goal

Confirm the quantities physically taken from the warehouse and deduct stock when the order preparation is complete.

## User

**Picker / Operator** (`operario`)

## Description

As an operator, I want to confirm the picked quantities per picking line, so I can deduct inventory and move the order toward completed dispatch.

## Acceptance criteria

- [ ] I can confirm `quantity_picked` per picking line (≤ `quantity_to_pick`)
- [ ] When confirmed, `quantity_on_hand` is decreased, `quantity_reserved` is released, and a `stock_movement` of type `outbound` is created
- [ ] A partial pick leaves the order in `partial`
- [ ] All lines complete → order `completed`
- [ ] Movement recorded with user and timestamp
- [ ] **Does not require** supervisor approval for normal confirmation

## Business rules

1. `quantity_picked` ≥ 0.
2. Do not confirm more than `quantity_to_pick` without a correction flow.
3. Atomic deduction via `StockUpdater`.
4. A `completed` order means all outbound lines have `picked_quantity` = `requested_quantity`, or an explicit partial close (MVP: allow a partial close with status `partial`).

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Pick 0 on a line | Mark the line as not supplied; partial order |
| Pick less than assigned | Release the excess reserved |
| Physical stock lower than the system at pick time | Block; force an adjustment via count |
| Double confirmation of the same line | Idempotent or reject the second confirmation |

## Preliminary technical notes

- `Picking::ConfirmLine.call(picking_line, quantity, user)` service
- Transaction: picking line + outbound line + StockUpdater
- UI optimized for tablet
- Branch: `feature/warehouse-picking`

## Versión en español

# US-022: Confirmar picking

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-021

---

## Objetivo

Confirmar las cantidades físicamente extraídas del almacén y descontar stock al completar la preparación del pedido.

## Usuario

**Picker / Operario** (`operario`)

## Descripción

Como operario, quiero confirmar las cantidades pickeadas por línea de picking, para descontar el inventario y avanzar el pedido hacia despacho completado.

## Criterios de aceptación

- [ ] Puedo confirmar `quantity_picked` por picking line (≤ `quantity_to_pick`)
- [ ] Al confirmar, se decrementa `quantity_on_hand`, se libera `quantity_reserved` y se crea `stock_movement` tipo `outbound`
- [ ] Picking parcial deja orden en `partial`
- [ ] Todas las líneas completas → orden `completed`
- [ ] Movimiento registrado con usuario y timestamp
- [ ] **No requiere aprobación** de supervisor para confirmación normal

## Reglas de negocio

1. `quantity_picked` ≥ 0.
2. No confirmar más de `quantity_to_pick` sin flujo de corrección.
3. Descuento atómico vía `StockUpdater`.
4. Orden `completed` implica todas las líneas de outbound con `picked_quantity` = `requested_quantity` o cierre explícito parcial (MVP: permitir cierre parcial con estado `partial`).

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Pickear 0 en línea | Marcar línea como no surtida; orden parcial |
| Pickear menos de lo asignado | Liberar exceso reservado |
| Stock físico menor al sistema en momento pick | Bloquear; forzar ajuste por conteo |
| Doble confirmación misma línea | Idempotente o rechazar segunda confirmación |

## Notas técnicas preliminares

- Servicio `Picking::ConfirmLine.call(picking_line, quantity, user)`
- Transacción: picking line + outbound line + StockUpdater
- UI optimizada para tablet
- Rama: `feature/warehouse-picking`
