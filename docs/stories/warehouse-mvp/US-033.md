# US-033: Approve or reject inventory adjustments

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-032

---

## Goal

Ensure supervisory control over stock changes that are not standard receiving, shipping, or transfer.

## User

**Supervisor** (`supervisor`), **Administrator** (`admin`)

## Description

As a supervisor, I want to review, approve, or reject manual adjustment requests and count corrections, so only validated changes affect the inventory.

## Acceptance criteria

- [ ] I see a queue of `pending` requests with: requester, date, type, warehouse, lines, reason
- [ ] I can **approve**: applies changes via `StockUpdater`, creates a `stock_movement` of type `adjustment`, status → `approved`
- [ ] I can **reject** with a required comment; status → `rejected`; no stock change
- [ ] I cannot approve my own request if I am the one who created it (separation of duties)
- [ ] Approval is atomic: all lines or none
- [ ] Decision history is available

## Business rules

1. Only `supervisor` and `admin` can approve.
2. Admin can approve their own requests (operational override).
3. After approval, the request is immutable.
4. The audit movement includes a reference to the `inventory_adjustment_id`.
5. Linked count corrections close the count as `closed` upon approval.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Stock changed between request and approval | Re-validate; reject if the result is negative |
| Approve with reservations that block it | Reject with a message |
| Reject the entire count | The count returns to `in_progress` or `cancelled` (MVP: `in_progress` for re-count) |
| Timeout on an old request | No auto-expiration in the MVP |

## Preliminary technical notes

- `Adjustments::Approve.call(adjustment, approver)` service and `#reject`
- Multi-line transaction + StockUpdater
- UI queue: `/inventory/adjustments/pending`
- Branch: `feature/warehouse-adjustments-approval`

## Versión en español

# US-033: Aprobar o rechazar ajustes de inventario

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-032

---

## Objetivo

Garantizar control supervisorio sobre cambios de stock que no son recepción, salida ni transferencia estándar.

## Usuario

**Supervisor** (`supervisor`), **Administrador** (`admin`)

## Descripción

Como supervisor, quiero revisar, aprobar o rechazar solicitudes de ajuste manual y correcciones de conteo, para que solo cambios validados impacten el inventario.

## Criterios de aceptación

- [ ] Veo cola de solicitudes `pending` con: solicitante, fecha, tipo, almacén, líneas, motivo
- [ ] Puedo **aprobar**: aplica cambios vía `StockUpdater`, crea `stock_movement` tipo `adjustment`, estado → `approved`
- [ ] Puedo **rechazar** con comentario obligatorio; estado → `rejected`; sin cambio de stock
- [ ] No puedo aprobar mi propia solicitud si soy quien la creó (segregación de funciones)
- [ ] Aprobación es atómica: todas las líneas o ninguna
- [ ] Historial de decisiones consultable

## Reglas de negocio

1. Solo `supervisor` y `admin` aprueban.
2. Admin puede aprobar propias solicitudes (override operativo).
3. Post-aprobación, solicitud es inmutable.
4. Movimiento de auditoría incluye referencia al `inventory_adjustment_id`.
5. Correcciones de conteo vinculadas cierran el conteo como `closed` al aprobar.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Stock cambió entre solicitud y aprobación | Re-validar; rechazar si resultante negativo |
| Aprobar con reservas que bloquean | Rechazar con mensaje |
| Rechazar conteo completo | Conteo vuelve a `in_progress` o `cancelled` (MVP: `in_progress` para re-conteo) |
| Timeout solicitud antigua | Sin auto-expiración en MVP |

## Notas técnicas preliminares

- Servicio `Adjustments::Approve.call(adjustment, approver)` y `#reject`
- Transacción multi-línea + StockUpdater
- Cola UI: `/inventory/adjustments/pending`
- Rama: `feature/warehouse-adjustments-approval`
