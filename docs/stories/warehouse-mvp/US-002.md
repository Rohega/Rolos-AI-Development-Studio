# US-002: Product management

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-001

---

## Goal

Keep the master product catalog that the distributor moves between warehouses, identifying products by SKU and allowing flexible search without depending on a barcode.

## User

**Administrator** (`admin`)

## Description

As an administrator, I want to create and edit products with a SKU, name, category, and unit of measure, and optionally a barcode, so that operators and supervisors can identify them during receiving, shipping, and lookups.

## Acceptance criteria

- [ ] I can create a product with: SKU (required), name, category, unit of measure
- [ ] Unit of measure only accepts: `unidad`, `caja`, `paquete`, `kg`, `litro`
- [ ] I can assign an optional `barcode`; if provided, it must be unique in the system
- [ ] I cannot duplicate a SKU
- [ ] I can deactivate a product; it does not appear in new receiving/shipping/transfer orders
- [ ] I can search for products by SKU, name (partial), or category
- [ ] The list shows SKU, name, category, unit, status, and barcode if present

## Business rules

1. SKU is globally unique and cannot be changed after creation (not editable in the MVP).
2. `barcode` is nullable; if present, it is globally unique.
3. Inactive product: existing stock remains; no new operations are allowed.
4. No conversions between units: every quantity operates in the product's `unit_type`.
5. The category must be active when creating a new product.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Empty barcode vs null | Treated as having no barcode |
| Two products with the same barcode | Rejected during validation |
| Deactivate a product with stock > 0 | Allowed with a warning |
| Search by name with accents | Case-insensitive search, basic normalization |
| SKU with leading/trailing spaces | Automatic trim |

## Preliminary technical notes

- `Product` model: `sku`, `name`, `category_id`, `unit_type` (enum), `barcode` (nullable, unique), `active`
- `unit_type` enum validation in the model and API
- Prepare `has_many :external_references` for the v2 ERP
- Indexes: `sku` (unique), `barcode` (unique, nulls allowed), `name` (fulltext or index for LIKE)
- API: `GET/POST/PATCH /api/products` with filters `q`, `category_id`, `active`

## Versión en español

# US-002: Gestión de productos

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-001

---

## Objetivo

Mantener el catálogo maestro de productos que la distribuidora mueve entre almacenes, con identificación por SKU y búsqueda flexible sin depender de código de barras.

## Usuario

**Administrador** (`admin`)

## Descripción

Como administrador, quiero crear y editar productos con SKU, nombre, categoría y unidad de medida, y opcionalmente un código de barras, para que operarios y supervisores puedan identificarlos en recepción, salida y consultas.

## Criterios de aceptación

- [ ] Puedo crear un producto con: SKU (obligatorio), nombre, categoría, unidad de medida
- [ ] Unidad de medida solo admite: `unidad`, `caja`, `paquete`, `kg`, `litro`
- [ ] Puedo asignar `barcode` opcional; si se informa, debe ser único en el sistema
- [ ] No puedo duplicar SKU
- [ ] Puedo desactivar un producto; no aparece en nuevas órdenes de recepción/salida/transferencia
- [ ] Puedo buscar productos por SKU, nombre (parcial) o categoría
- [ ] El listado muestra SKU, nombre, categoría, unidad, estado y barcode si existe

## Reglas de negocio

1. SKU único global e inmutable tras creación (no editable en MVP).
2. `barcode` nullable; si presente, único global.
3. Producto inactivo: stock existente permanece; no nuevas operaciones.
4. Sin conversiones entre unidades: toda cantidad opera en `unit_type` del producto.
5. Categoría debe estar activa al crear producto nuevo.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Barcode vacío vs null | Tratados como sin barcode |
| Dos productos con mismo barcode | Rechazado en validación |
| Desactivar producto con stock > 0 | Permitido con advertencia |
| Búsqueda por nombre con acentos | Búsqueda case-insensitive, normalización básica |
| SKU con espacios al inicio/fin | Trim automático |

## Notas técnicas preliminares

- Modelo `Product`: `sku`, `name`, `category_id`, `unit_type` (enum), `barcode` (nullable, unique), `active`
- Validación enum `unit_type` en modelo y API
- Preparar `has_many :external_references` para v2 ERP
- Índices: `sku` (unique), `barcode` (unique, nulls allowed), `name` (fulltext o índice para LIKE)
- API: `GET/POST/PATCH /api/products` con filtros `q`, `category_id`, `active`
