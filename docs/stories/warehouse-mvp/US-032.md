# US-032: Request a stock adjustment or correction

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-031

---

## Goal

Formalize inventory change requests that do not come from the normal receiving/shipping/transfer flow, including count corrections and manual adjustments.

## User

**Supervisor** (`supervisor`), **Operator** (`operario`)

## Description

As a supervisor or operator, I want to request a manual stock adjustment or submit corrections detected in a count, so a supervisor approves them before the inventory is modified.

## Acceptance criteria

- [ ] **Manual adjustment:** I can create a request with product, location, target quantity or delta, and a required reason
- [ ] **Count correction:** when submitting a count (US-031), an `InventoryAdjustment` is created in `pending` with lines for each variance ≠ 0
- [ ] A request in `pending` does not modify stock
- [ ] I can list my pending requests
- [ ] Reason is required (min. 10 characters)
- [ ] An operator can request; they cannot self-approve

## Business rules

1. Any stock change outside the standard operational flows requires this flow.
2. The resulting quantity cannot be negative after the adjustment.
3. A manual request affects one or more lines from the same warehouse.
4. `source_type`: `manual` | `count`.
5. `rejected` requests do not apply changes; they stay in the history.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Adjust to 0 (empty a location) | Allowed if there are no pending reservations |
| Adjust with active reservations | Reject if the result < reserved |
| Double request on the same pending location | Reject the second until the first is resolved |
| Count correction with a large negative variance | Allow it; the supervisor validates at approval |

## Preliminary technical notes

- `InventoryAdjustment`, `InventoryAdjustmentLine` models
- `Adjustments::RequestManual`, `Adjustments::FromCount` services
- States: `pending`, `approved`, `rejected`
- Simple notification to supervisors (email/UI badge P1)
- Branch: `feature/warehouse-counts`

## Versión en español

# US-032: Solicitar ajuste o corrección de stock

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-031

---

## Objetivo

Formalizar solicitudes de cambio de inventario que no provienen de recepción/salida/transferencia normal, incluyendo correcciones por conteo y ajustes manuales.

## Usuario

**Supervisor** (`supervisor`), **Operario** (`operario`)

## Descripción

Como supervisor u operario, quiero solicitar un ajuste manual de stock o enviar correcciones detectadas en un conteo, para que un supervisor apruebe antes de modificar el inventario.

## Criterios de aceptación

- [ ] **Ajuste manual:** puedo crear solicitud con producto, ubicación, cantidad objetivo o delta, y motivo obligatorio
- [ ] **Corrección por conteo:** al enviar conteo (US-031), se crea `InventoryAdjustment` en `pending` con líneas por varianza ≠ 0
- [ ] Solicitud en `pending` no modifica stock
- [ ] Puedo listar mis solicitudes pendientes
- [ ] Motivo/reason obligatorio (mín. 10 caracteres)
- [ ] Operario puede solicitar; no puede auto-aprobar

## Reglas de negocio

1. Toda modificación de stock fuera de flujos operativos estándar requiere este flujo.
2. Cantidad resultante no puede ser negativa tras ajuste.
3. Una solicitud manual afecta una o más líneas del mismo almacén.
4. `source_type`: `manual` | `count`.
5. Solicitudes `rejected` no aplican cambios; quedan en historial.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Ajuste a 0 (agotar ubicación) | Permitido si no hay reservas pendientes |
| Ajuste con reservas activas | Rechazar si resultante < reservado |
| Doble solicitud misma ubicación pendiente | Rechazar segunda hasta resolver primera |
| Corrección conteo con varianza negativa grande | Permitir; supervisor valida en aprobación |

## Notas técnicas preliminares

- Modelos `InventoryAdjustment`, `InventoryAdjustmentLine`
- Servicios `Adjustments::RequestManual`, `Adjustments::FromCount`
- Estados: `pending`, `approved`, `rejected`
- Notificación simple a supervisores (email/UI badge P1)
- Rama: `feature/warehouse-counts`
