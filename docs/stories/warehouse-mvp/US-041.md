# US-041: Inventory movement history

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-005

---

## Goal

Investigate discrepancies and audit operations by consulting the immutable record of stock changes per product.

## User

**Supervisor** (`supervisor`), **Administrator** (`admin`), **Viewer** (`consulta` — read only)

## Description

As a supervisor, I want to consult a product's movement history filtering by warehouse, type, and date range, so I can investigate differences and meet operational traceability.

## Acceptance criteria

- [ ] Paginated list of `stock_movements` filterable by: product, warehouse, type, user, date range
- [ ] Each record shows: date, type, quantity (+/-), before/after, location, user, reference (source order), notes
- [ ] Cancelled movements show a link to the reversal and a "cancelado" badge
- [ ] CSV export of the current filter
- [ ] Records are read only in the UI (no editing)
- [ ] Sorted by `occurred_at` descending

## Business rules

1. `stock_movements` is append-only; no update/delete from the application.
2. Every mutation via `StockUpdater` generates exactly one movement (except multi-location operations: one movement per line).
3. The viewer can see the entire history; no restricted export in the MVP.
4. Retention: unlimited in the MVP.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Product with no movements | Empty list with a message |
| Transfer movement | Show the linked out/in pair visually (P1); MVP: type and reference |
| High volume (>100k movements) | Pagination + indexes; no full load |
| Invalid date filter | Validation |

## Preliminary technical notes

- `stock_movements` table with indexes `(product_id, occurred_at)`, `(warehouse_id, occurred_at)`
- Polymorphic `reference_type`/`reference_id`
- API: `GET /api/stock_movements?product_id=&warehouse_id=&...`
- UI at `/products/:id/movements` and global view `/audit/movements`
- Branch: `feature/warehouse-stock-core` (API) + `feature/warehouse-audit-ui` (full UI)

## Versión en español

# US-041: Historial de movimientos de inventario

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-005

---

## Objetivo

Investigar discrepancias y auditar operaciones consultando el registro inmutable de cambios de stock por producto.

## Usuario

**Supervisor** (`supervisor`), **Administrador** (`admin`), **Consulta** (`consulta` — solo lectura)

## Descripción

Como supervisor, quiero consultar el historial de movimientos de un producto filtrando por almacén, tipo y rango de fechas, para investigar diferencias y cumplir trazabilidad operativa.

## Criterios de aceptación

- [ ] Listado paginado de `stock_movements` filtrable por: producto, almacén, tipo, usuario, rango fechas
- [ ] Cada registro muestra: fecha, tipo, cantidad (+/-), antes/después, ubicación, usuario, referencia (orden origen), notas
- [ ] Movimientos cancelados muestran enlace a reversión y badge "cancelado"
- [ ] Export CSV del filtro actual
- [ ] Registros son de solo lectura en UI (sin edición)
- [ ] Ordenación por `occurred_at` descendente

## Reglas de negocio

1. `stock_movements` append-only; sin update/delete desde aplicación.
2. Toda mutación vía `StockUpdater` genera exactamente un movimiento (salvo operaciones multi-ubicación: un movimiento por línea).
3. Consulta puede ver todo el historial; no export restringido en MVP.
4. Retención: ilimitada en MVP.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Producto sin movimientos | Lista vacía con mensaje |
| Movimiento de transferencia | Mostrar par out/in vinculado visualmente (P1); MVP: tipo y referencia |
| Alto volumen (>100k movimientos) | Paginación + índices; sin carga total |
| Filtro fecha inválida | Validación |

## Notas técnicas preliminares

- Tabla `stock_movements` con índices `(product_id, occurred_at)`, `(warehouse_id, occurred_at)`
- `reference_type`/`reference_id` polimórfico
- API: `GET /api/stock_movements?product_id=&warehouse_id=&...`
- UI en `/products/:id/movements` y vista global `/audit/movements`
- Rama: `feature/warehouse-stock-core` (API) + `feature/warehouse-audit-ui` (UI completa)
