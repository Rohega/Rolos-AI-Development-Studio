# US-005: Initial stock load

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-002, US-004

---

## Goal

Set up the starting inventory in each warehouse when launching the system from scratch, without an ERP integration.

## User

**Administrator** (`admin`)

## Description

As an administrator, I want to load initial product quantities by location and warehouse, using an individual form or a CSV import, so I can start operating with real stock from day one.

## Acceptance criteria

- [ ] I can assign an initial quantity to a product + location combination
- [ ] The load creates or updates a `stock_level` and records a `stock_movement` of type `adjustment` with the note "stock inicial"
- [ ] I can import a CSV with the columns: `warehouse_code`, `location_code`, `sku`, `quantity`
- [ ] The import reports valid and invalid rows without partially applying an invalid batch (all-or-nothing per file)
- [ ] Quantity must be ≥ 0
- [ ] Product and location must exist and be active
- [ ] The location must belong to the indicated warehouse

## Business rules

1. Initial load is only allowed if the current `quantity_on_hand` is 0 for that combination, unless there is an explicit re-import with the `overwrite` flag (admin only).
2. Every load goes through `StockUpdater`.
3. Units follow the product's `unit_type`; no conversion.
4. Audit movement with the `user_id` of the admin doing the load.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| SKU not found in the CSV | Error on the row; rollback of the entire import |
| Decimal quantity for a `unidad` product | Allow decimals only for `kg` and `litro`; integers recommended for the rest |
| Same combination duplicated in the CSV | Reject the file with a duplicate error |
| Location from another warehouse | Reject the row |
| Stock already exists without overwrite | Reject the row with a message |

## Preliminary technical notes

- `StockUpdater#apply_initial_stock(product, location, quantity, user)` service
- Optional job for large imports (>1000 rows)
- CSV template downloadable from the UI
- DB transaction per import file
- Branch: `feature/warehouse-stock-core`

## Versión en español

# US-005: Carga de stock inicial

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-002, US-004

---

## Objetivo

Establecer el inventario inicial en cada almacén al arrancar el sistema en blanco, sin integración ERP.

## Usuario

**Administrador** (`admin`)

## Descripción

Como administrador, quiero cargar cantidades iniciales de productos por ubicación y almacén, mediante formulario individual o importación CSV, para comenzar a operar con stock real desde el día uno.

## Criterios de aceptación

- [ ] Puedo asignar cantidad inicial a combinación producto + ubicación
- [ ] La carga crea o actualiza `stock_level` y registra `stock_movement` tipo `adjustment` con nota "stock inicial"
- [ ] Puedo importar CSV con columnas: `warehouse_code`, `location_code`, `sku`, `quantity`
- [ ] El import reporta filas válidas e inválidas sin aplicar parcialmente un lote inválido (todo-o-nada por archivo)
- [ ] Cantidad debe ser ≥ 0
- [ ] Producto y ubicación deben existir y estar activos
- [ ] Ubicación debe pertenecer al almacén indicado

## Reglas de negocio

1. Carga inicial solo permitida si `quantity_on_hand` actual es 0 para esa combinación, salvo re-import explícito con flag `overwrite` (solo admin).
2. Toda carga pasa por `StockUpdater`.
3. Unidades según `unit_type` del producto; sin conversión.
4. Movimiento de auditoría con `user_id` del admin que carga.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| SKU inexistente en CSV | Error en fila; rollback del import completo |
| Cantidad decimal en producto `unidad` | Permitir decimales solo para `kg` y `litro`; enteros recomendados para resto |
| Misma combinación duplicada en CSV | Rechazar archivo con error de duplicado |
| Ubicación de otro almacén | Rechazar fila |
| Stock ya existente sin overwrite | Rechazar fila con mensaje |

## Notas técnicas preliminares

- Servicio `StockUpdater#apply_initial_stock(product, location, quantity, user)`
- Job opcional para imports grandes (>1000 filas)
- Template CSV descargable desde UI
- Transacción DB por archivo de import
- Rama: `feature/warehouse-stock-core`
