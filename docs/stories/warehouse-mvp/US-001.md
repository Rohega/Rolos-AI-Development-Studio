# US-001: Product category management

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** none

---

## Goal

Allow the product catalog to be organized into categories to make searching, filtering, and reporting easier in a distribution operation.

## User

**Administrator** (`admin`)

## Description

As an administrator, I want to create, edit, and deactivate product categories, so I can group the catalog and enable category searches in inventory and operations.

## Acceptance criteria

- [ ] I can create a category with a required name
- [ ] I can assign an optional parent category (one level of hierarchy)
- [ ] I cannot create two active categories with the same name
- [ ] I can deactivate a category; inactive categories do not appear when creating new products
- [ ] A category with active products cannot be physically deleted; it can only be deactivated
- [ ] Changes are recorded with creation/update timestamps

## Business rules

1. Category name must be unique among active categories.
2. Maximum hierarchy of 1 level: parent → child in the MVP.
3. Deactivating a parent category does not automatically deactivate its children.
4. Products from a deactivated category still exist and can still be queried.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Deactivate a category with active products | Allowed; products keep their `category_id` |
| Create a child category under an inactive category | Rejected with a clear message |
| Rename a category to an existing name | Rejected |
| Delete a category with no products | Soft-delete (deactivate) in the MVP; no hard delete |

## Preliminary technical notes

- `Category` model: `name`, `parent_id` (nullable FK self), `active` (default true)
- Partial unique index or app-level validation on `name` where `active = true`
- API: `GET/POST/PATCH /api/categories`
- Seed: example categories (`Alimentación`, `Bebidas`, `Limpieza`)

## Versión en español

# US-001: Gestión de categorías de producto

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** ninguna

---

## Objetivo

Permitir organizar el catálogo de productos en categorías para facilitar búsqueda, filtrado y reportes en una operación de distribución.

## Usuario

**Administrador** (`admin`)

## Descripción

Como administrador, quiero crear, editar y desactivar categorías de producto, para agrupar el catálogo y habilitar búsquedas por categoría en inventario y operaciones.

## Criterios de aceptación

- [ ] Puedo crear una categoría con nombre obligatorio
- [ ] Puedo asignar una categoría padre opcional (un nivel de jerarquía)
- [ ] No puedo crear dos categorías activas con el mismo nombre
- [ ] Puedo desactivar una categoría; las categorías inactivas no aparecen al crear productos nuevos
- [ ] Una categoría con productos activos no se puede eliminar físicamente; solo desactivar
- [ ] Los cambios quedan registrados con timestamp de creación/actualización

## Reglas de negocio

1. Nombre de categoría único entre categorías activas.
2. Jerarquía máxima de 1 nivel padre → hijo en MVP.
3. Desactivar categoría padre no desactiva automáticamente las hijas.
4. Productos de categoría desactivada siguen existiendo y siendo consultables.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Desactivar categoría con productos activos | Permitido; productos conservan `category_id` |
| Crear categoría hija de categoría inactiva | Rechazado con mensaje claro |
| Renombrar categoría a nombre existente | Rechazado |
| Eliminar categoría sin productos | Soft-delete (desactivar) en MVP; no hard delete |

## Notas técnicas preliminares

- Modelo `Category`: `name`, `parent_id` (nullable FK self), `active` (default true)
- Índice único parcial o validación app-level en `name` donde `active = true`
- API: `GET/POST/PATCH /api/categories`
- Semilla: categorías ejemplo (`Alimentación`, `Bebidas`, `Limpieza`)
