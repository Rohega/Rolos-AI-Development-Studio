# US-020: Create an outbound order

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-002, US-003

---

## Goal

Record outbound goods orders from a warehouse to the distributor's customers, preparing the picking flow.

## User

**Operator** (`operario`), **Supervisor** (`supervisor`), **Administrator** (`admin`)

## Description

As an operator, I want to create an outbound order with a source warehouse, customer (text), reference, and product lines with requested quantities, so I can prepare the dispatch without needing supervisor approval.

## Acceptance criteria

- [ ] I can create an order in `draft` with: warehouse, `customer_name`, `reference_number` (optional), lines (product, quantity)
- [ ] Only active products and warehouses
- [ ] When moving to `picking`, the system validates availability (see US-023)
- [ ] I can cancel an order in `draft` with no impact on stock
- [ ] Auto-generated internal number (e.g. `OUT-2026-00001`)
- [ ] Creating and releasing to picking **does not require** supervisor approval

## Business rules

1. A `draft` order does not reserve stock.
2. When picking starts, stock is reserved (`quantity_reserved`).
3. `customer_name` is free text in the MVP.
4. Requested quantity > 0.
5. An order belongs to a single source warehouse.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Start picking without enough stock | Block or allow partial per US-023 |
| Deactivated product in the draft | Warning; do not release to picking |
| Cancel an order in `picking` | Release reservations; status `cancelled` |
| Multiple lines with the same product | Allow with a warning |

## Preliminary technical notes

- `OutboundOrder`, `OutboundLine` models
- States: `draft`, `picking`, `partial`, `completed`, `cancelled`
- `ReservationService#reserve(order)` when moving to picking
- API: `POST /api/warehouses/:id/outbound_orders`
- Branch: `feature/warehouse-outbound`
