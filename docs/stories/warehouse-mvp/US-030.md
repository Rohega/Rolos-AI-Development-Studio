# US-030: Multi-warehouse inventory lookup

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-005

---

## Goal

Give sales, purchasing, and management a consolidated and per-warehouse view of stock across the distributor.

## User

**Viewer** (`consulta`), **Supervisor** (`supervisor`), **Administrator** (`admin`), **Operator** (`operario` — read only)

## Description

As a viewer, I want to search inventory by SKU, name, or category and filter by warehouse or see consolidated totals, so I can quickly answer questions about product availability.

## Acceptance criteria

- [ ] Search by SKU (exact), name (partial), or category
- [ ] Filter by a specific warehouse or an "all warehouses" view
- [ ] Result shows: product, category, unit, available quantity, reserved quantity, breakdown by location (expandable)
- [ ] Response time < 3 s with up to 10,000 product-location combinations
- [ ] CSV export of the visible result
- [ ] The `consulta` role is read only; no modification buttons

## Business rules

1. Displayed available = `on_hand - reserved` per location; product total = sum within the warehouse.
2. Inactive products shown with an "inactivo" badge.
3. Inactive warehouses shown in history but flagged.
4. No barcode required for search.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Product with no stock | Show with quantities of 0 |
| Empty search | List paginated with the applied filters |
| Same SKU in several warehouses | Rows grouped by warehouse |
| Parent category selected | Include products from subcategories |

## Preliminary technical notes

- Endpoint: `GET /api/inventory?warehouse_id=&category_id=&q=`
- Indexes: `(warehouse_id, product_id)` on `stock_levels`
- Pagination required (50 per page)
- Inertia view with an expandable table
- Branch: `feature/warehouse-inventory-query`

## Versión en español

# US-030: Consulta de inventario multi-almacén

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-005

---

## Objetivo

Dar visibilidad consolidada y por almacén del stock a ventas, compras y dirección de la distribuidora.

## Usuario

**Consulta** (`consulta`), **Supervisor** (`supervisor`), **Administrador** (`admin`), **Operario** (`operario` — solo lectura)

## Descripción

Como usuario de consulta, quiero buscar inventario por SKU, nombre o categoría y filtrar por almacén o ver totales consolidados, para responder rápidamente sobre disponibilidad de productos.

## Criterios de aceptación

- [ ] Búsqueda por SKU (exacto), nombre (parcial) o categoría
- [ ] Filtro por almacén específico o vista "todos los almacenes"
- [ ] Resultado muestra: producto, categoría, unidad, cantidad disponible, cantidad reservada, desglose por ubicación (expandible)
- [ ] Tiempo de respuesta < 3 s con hasta 10.000 combinaciones producto-ubicación
- [ ] Export CSV del resultado visible
- [ ] Rol `consulta` solo lectura; sin botones de modificación

## Reglas de negocio

1. Disponible mostrado = `on_hand - reserved` por ubicación; total producto = suma en almacén.
2. Productos inactivos visibles con badge "inactivo".
3. Almacenes inactivos visibles en histórico pero marcados.
4. Sin código de barras obligatorio para búsqueda.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Producto sin stock | Mostrar con cantidades 0 |
| Búsqueda vacía | Listar paginado con filtros aplicados |
| Mismo SKU en varios almacenes | Filas agrupadas por almacén |
| Categoría padre seleccionada | Incluir productos de subcategorías |

## Notas técnicas preliminares

- Endpoint: `GET /api/inventory?warehouse_id=&category_id=&q=`
- Índices: `(warehouse_id, product_id)` en `stock_levels`
- Paginación obligatoria (50 por página)
- Vista Inertia con tabla expandible
- Rama: `feature/warehouse-inventory-query`
