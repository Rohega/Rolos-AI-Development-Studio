# US-010: Create a receiving order

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-002, US-003

---

## Goal

Plan incoming supplier goods into a specific warehouse before physically confirming the receipt.

## User

**Operator** (`operario`), **Supervisor** (`supervisor`), **Administrator** (`admin`)

## Description

As an operator, I want to create a receiving order specifying the warehouse, supplier (text), reference, and product lines with expected quantities, so I can prepare incoming goods at the distribution center.

## Acceptance criteria

- [ ] I can create an order in `draft` status with: warehouse, `supplier_name`, `reference_number` (optional), lines (product, expected quantity)
- [ ] Only active products are available in the selector
- [ ] Only active warehouses
- [ ] I can save a draft and edit lines before confirming the receipt
- [ ] I can cancel an order in `draft` with no impact on stock
- [ ] The order shows an auto-generated internal number (e.g. `REC-2026-00001`)

## Business rules

1. An order in `draft` does not modify stock.
2. At least one line is required to move to confirmation (US-011).
3. `supplier_name` is free text in the MVP (no supplier catalog).
4. Expected quantity > 0.
5. Implicit unit: the product's `unit_type`.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Order with no lines when saving | Allow an empty draft with a warning |
| Product deactivated after creating the draft | Show a warning; cannot be confirmed until the line is removed |
| Same product on two lines | Allow it (different batches in the future); MVP: warn about the duplicate |
| Warehouse deactivated before confirming | Block confirmation |

## Preliminary technical notes

- `ReceptionOrder`, `ReceptionLine` models
- States: `draft`, `partial`, `completed`, `cancelled`
- `reference_number` indexed for search
- API: `POST /api/warehouses/:id/reception_orders`, `PATCH` for the draft
- Branch: `feature/warehouse-reception`

## Versión en español

# US-010: Crear orden de recepción

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-002, US-003

---

## Objetivo

Planificar la entrada de mercancía de proveedores en un almacén concreto antes de confirmar físicamente la recepción.

## Usuario

**Operario** (`operario`), **Supervisor** (`supervisor`), **Administrador** (`admin`)

## Descripción

Como operario, quiero crear una orden de recepción indicando almacén, proveedor (texto), referencia y líneas de productos con cantidades esperadas, para preparar la entrada de mercancía al centro de distribución.

## Criterios de aceptación

- [ ] Puedo crear orden en estado `draft` con: almacén, `supplier_name`, `reference_number` (opcional), líneas (producto, cantidad esperada)
- [ ] Solo productos activos disponibles en el selector
- [ ] Solo almacenes activos
- [ ] Puedo guardar borrador y editar líneas antes de confirmar recepción
- [ ] Puedo cancelar orden en `draft` sin impacto en stock
- [ ] La orden muestra número interno autogenerado (ej. `REC-2026-00001`)

## Reglas de negocio

1. Orden en `draft` no modifica stock.
2. Al menos una línea para pasar a confirmación (US-011).
3. `supplier_name` es texto libre en MVP (sin catálogo de proveedores).
4. Cantidad esperada > 0.
5. Unidad implícita: `unit_type` del producto.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Orden sin líneas al guardar | Permitir borrador vacío con advertencia |
| Producto desactivado tras crear borrador | Mostrar advertencia; no confirmable hasta quitar línea |
| Mismo producto en dos líneas | Permitir (lotes distintos futuro); MVP: advertir duplicado |
| Almacén desactivado antes de confirmar | Bloquear confirmación |

## Notas técnicas preliminares

- Modelos `ReceptionOrder`, `ReceptionLine`
- Estados: `draft`, `partial`, `completed`, `cancelled`
- `reference_number` indexado para búsqueda
- API: `POST /api/warehouses/:id/reception_orders`, `PATCH` para borrador
- Rama: `feature/warehouse-reception`
