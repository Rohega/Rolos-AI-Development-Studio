# US-031: Inventory count

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Priority:** P0  
**Dependencies:** US-005

---

## Goal

Verify physical quantities in the warehouse and detect differences against the stock recorded in the system.

## User

**Supervisor** (`supervisor`), **Operator** (`operario`)

## Description

As a supervisor, I want to start an inventory count by warehouse (by location or by product) and record the physical quantities counted, so I can identify discrepancies that need an approved correction.

## Acceptance criteria

- [ ] I can create a count in `in_progress` status for a warehouse
- [ ] I can add lines: product + location + counted quantity
- [ ] The system shows the current `system_quantity` and `variance` = counted − system
- [ ] I can save progress without sending it for approval
- [ ] When submitted (`submitted`), a linked correction request is generated (US-032)
- [ ] Only one `in_progress` count per warehouse at a time

## Business rules

1. A count does not modify stock until the adjustment is approved (US-033).
2. Lines with a variance of 0 do not generate an adjustment.
3. Location and product must exist in the warehouse.
4. An operator can record counts; submitting for approval requires `supervisor` or `operario` depending on policy → **MVP: operator records; supervisor submits/closes**.

## Edge cases

| Case | Expected behavior |
|------|-------------------------|
| Stock changes during the count | `system_quantity` captured when creating the line or when submitting → **MVP: snapshot on submit** |
| Line with no physical count | Do not include it in the submission |
| Abandoned count | Cancel the count with no effect |
| Inactive product with stock | Allow counting |

## Preliminary technical notes

- `InventoryCount`, `InventoryCountLine` models
- `InventoryCount::Submit.call(count, user)` service creates a pending `InventoryAdjustment`
- Count states: `in_progress`, `submitted`, `closed`, `cancelled`
- Branch: `feature/warehouse-counts`

## Versión en español

# US-031: Conteo de inventario

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-005

---

## Objetivo

Verificar cantidades físicas en el almacén y detectar diferencias respecto al stock registrado en el sistema.

## Usuario

**Supervisor** (`supervisor`), **Operario** (`operario`)

## Descripción

Como supervisor, quiero iniciar un conteo de inventario por almacén (por ubicación o por producto) y registrar cantidades físicas contadas, para identificar discrepancias que requieran corrección aprobada.

## Criterios de aceptación

- [ ] Puedo crear conteo en estado `in_progress` para un almacén
- [ ] Puedo agregar líneas: producto + ubicación + cantidad contada
- [ ] El sistema muestra `system_quantity` actual y `variance` = contado − sistema
- [ ] Puedo guardar progreso sin enviar a aprobación
- [ ] Al enviar (`submitted`), se genera solicitud de corrección vinculada (US-032)
- [ ] Solo un conteo `in_progress` por almacén simultáneo

## Reglas de negocio

1. Conteo no modifica stock hasta aprobación de ajuste (US-033).
2. Líneas con varianza 0 no generan ajuste.
3. Ubicación y producto deben existir en el almacén.
4. Operario puede registrar conteos; enviar a aprobación requiere `supervisor` o `operario` según política → **MVP: operario registra; supervisor envía/cierra**.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Stock cambia durante conteo | `system_quantity` capturado al crear línea o al enviar → **MVP: snapshot al enviar** |
| Línea sin conteo físico | No incluir en envío |
| Conteo abandonado | Cancelar conteo sin efecto |
| Producto inactivo con stock | Permitir contar |

## Notas técnicas preliminares

- Modelos `InventoryCount`, `InventoryCountLine`
- Servicio `InventoryCount::Submit.call(count, user)` crea `InventoryAdjustment` pendiente
- Estados count: `in_progress`, `submitted`, `closed`, `cancelled`
- Rama: `feature/warehouse-counts`
